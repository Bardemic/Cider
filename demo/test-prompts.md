# Test Prompts for Cider Demo

## Primary Demo Prompt (most reliable)
Build a simple counter app for iPhone. It should have a large number display in the center, a "+" button to increment, a "-" button to decrement, and a "Reset" button. Use a clean, modern design with SF Symbols.

## Alternative Prompts

### Todo List
Build a simple todo list app for iPhone. Users should be able to add todos with a text field and button, see a list of todos, and tap to mark them complete (with strikethrough). Include a delete swipe action.

### Color Palette
Build an app that displays a grid of colorful squares. When you tap a square, it shows the hex color code in a large label at the top. Include at least 12 different colors in a 3-column grid.

### Tip Calculator
Build a tip calculator app. It should have a text field for the bill amount, a segmented control for tip percentage (15%, 18%, 20%, 25%), and display the tip amount and total. Keep it simple and clean.

### Flashcard
Build a flashcard app with 5 pre-loaded cards about iOS development. Show the question, tap to flip and see the answer, swipe to go to the next card. Show card number (e.g., "3/5").

## Tips for Demo
- Start with "Build a simple counter app" — it's the most reliable
- If the first build fails, that's actually good for the demo — shows the error → fix → rebuild loop
- After the build succeeds, click "Screenshot" to show the running app
- The full pipeline (prompt → build → screenshot) takes ~2-3 minutes
